Introduction

Coronavirus disease 2019 (COVID-19), caused by severe acute respiratory syndrome coronavirus 2 (SARS-CoV-2), grew rapidly in scale and severity, and was officially declared as a pandemic on March 11th, 2020. In the United States, the country with the highest confirmed cases, the government has announced a number of policies, including shutting down schools and workplaces, promoting social distancing, advising people to wear masks in public areas, etc. The U.S. president Donald Trump also broadcasts his opinions towards the pandemic, whose words might have an impact on the public's attitude.
In this study, applying the state-of-art natural language processing model, we attempt to analyze how the overall sentiment of COVID-19 related tweets changes with important policy decisions or influential words by politicians.

Data and Preprocessing

Datasets

There are two data sources that we utilized in this project. We do analysis with our model using the processed selection of COVID-19 Twitter dataset (Release 12.0). To train and evaluate our model, we used SemEval-2017 Task 4A dataset.

Preprocessing

The raw data from COVID-19 Twitter dataset (Release 12.0) contains a total of 331M tweet ID entries. As shown in Figure, we first randomly sampled 10,000 tweet IDs for each day from March 19th to May 30th (74 continuous days). By querying the twitter server, we hydrated the selected tweet ids to get the text and user information associated. (“Hydrate a tweet ID” simply means to get the tweet text and user information for the tweet specified by the tweet ID). We then filtered out all tweets from users whose location information implies they are in the states.

Text Processing

For the purpose of boosting our model’s performance, we preprocessed the tweet texts in the two datasets using a pipeline as shown in Figure. To be more specific, we first convert emoji to text, expand common ASCII and abbreviated text emoticons into human-readable text. The second step is to remove URLs present, replace “@[username]” with just “someone”, and expand hashtags (e.g. “#ATwitterHashtag” into “a twitter hashtag”). In the third step, we use WordNet to lemmatize the tweet text, changing words in different parts of speech and plural forms into one common canonical form. These three steps aim to make sure our BERT/RoBERTa based embedding can comprehend as much useful information as possible.

Model

the RoBERTa Model

The RoBERTa (Robustly Optimized BERT Pretraining) model was proposed by Yinhan Liu et al in 2018. It builds on BERT (Bidirectional Encoder Representations from Transformers), which is a language representation model designed to pre-train deep bidirectional representations from unlabeled text. As a result, the pre-trained BERT model can be fine-tuned with just one additional output layer to create state-of-the-art models for a wide range of tasks. A video introduction of BERT can be found below.

Fine-tuning RoBERTa

We splitted the SemEval-2017 Task 4A Dataset into train, validation and test sets. The statistics is the following:

Dataset Negative Neutral Positive Total
Train 4856 14368 11689 30913 (70%)
Validation 1401 4045 3386 8832 (20%)
Test 669 2072 1675 4416 (10%)
Total 6926 20485 16750 44161

In our sentiment analysis task, we utilize the transformers library by huggingface which includes a RoBERTa/BERT model base in Pytorch. On top of the RoBERTa/BERT base, we trained two layers of GRU (Gated Recurrent Unit) to predict the probability for 3 sentiment classes: “Negative”, “Neutral” and “Positive”, with the cross-entropy loss function minimized on Stochastic Gradient Descent Optimizer. We train the model on the train set and select the model with the lowest validation loss (the value of the loss function on the validation set).

Performance

Our final model has the following performance, evaluated using the validation set and the test set:

Metric Value
Accuracy (validation) 70.1%
Micro F1 score (validation) Placeholder
